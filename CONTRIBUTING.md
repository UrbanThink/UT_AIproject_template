# Contributing

## Before opening a pull request

- Read `AGENTS.md` and `docs/ai/README.md` to understand the AI workspace.
- Read the relevant scoped instructions in `.github/instructions/`.
- Follow the repository conventions described in `.github/instructions/conventions.instructions.md`.

## Change expectations

- Keep changes minimal, coherent, and reviewable.
- Preserve existing patterns and architecture.
- Add or update tests when behavior changes.
- Update documentation when behavior, configuration, or usage changes.
- Run formatting, linting, and tests before pushing.

## Commit messages

Follow Conventional Commits: `type(scope): short description`

Common types: `feat`, `fix`, `docs`, `test`, `refactor`, `chore`, `ci`, `style`, `perf`.

## Pull request expectations

- Provide a clear title and description.
- Reference the related issue or task when applicable.
- Ensure CI checks pass.
- Request review from appropriate maintainers.

## AI-assisted contributions

AI-generated code follows the same quality, security, and review standards as human-written code. No exceptions.

## Security

If you discover a security vulnerability, do not open a public issue. Contact the maintainers privately.
